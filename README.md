# 执行器配置程序 TE-AP-00（Windows 桌面版）

本项目是把原 Excel/VBA 宏程序
《TE-AP-00_执行器配置小程序_-260413A0.xlsm》
整体重构为一个独立运行的 Windows 桌面程序（Python + Tkinter），
可以打包成单独的 `.exe`，不再依赖 Excel 也能使用。

---

## 一、这个程序能做什么

1. **配置算法**：根据阀门的"配置扭矩、上法兰尺寸、上连接轴尺寸、安全系数、
   气源压力"，自动算出 DA气动执行器 / SR气动执行器 / 精小型电动执行器 /
   Q型电动执行器 / 蜗轮，共 5 种执行器的推荐型号（含外形尺寸），以及
   DA / SR 对应的离合器配置。算法与原 Excel 宏逐一对应移植，
   结果保持一致（详见第四节"与原Excel的对比"）。
2. **版本记录**：见菜单【帮助 - 版本更新记录】，完整保留了原 Excel 中
   2025-08-15 至 2026-04-13 的全部历史记录，并续写了后续每一次调整的更新说明。
3. **密码保护**：执行器参数（型号、扭矩、尺寸等）需要管理员密码才能修改，
   入口在菜单【工具 - 厂家数据管理】。出厂默认密码见下方"密码说明"。
4. **Excel 复制/粘贴**：可以从 Excel 复制一段阀门参数表（几列几行都行），
   在程序里点【从剪贴板粘贴】导入；算完之后点【复制全部结果到剪贴板】，
   再回到 Excel 里粘贴即可。
5. **每种执行器类型可分别录入多个厂家数据**：DA气动执行器、SR气动执行器、
   精小型电动执行器、Q型电动执行器、蜗轮——这 5 种类型各自下面都可以
   挂多个厂家的型号数据库（比如 DA 气动执行器同时有"VATTEN"和"某某厂家"
   两份数据），并且可以在【工具 - 数据来源设置】里为每一种类型分别选择
   当前用哪个厂家的数据，5 种类型互不影响、可以来自不同厂家。
6. **填写区/输出区一眼区分**：表格左侧浅黄色是可以填写的阀门参数区，
   右侧浅蓝色是算出来的结果区（只读），不会分不清哪里能改、哪里是结果。
7. **输出列可自由增减显示**：在【工具 - 显示列设置】里勾选只看哪几种
   执行器类型的结果（比如只看DA气动执行器），复制到Excel时也只复制
   勾选显示的部分。
8. **型号/连接尺寸变体自动查重**：在厂家数据编辑器里新增或修改型号名称、
   连接尺寸变体时，如果出现重复，会自动标红高亮并提示，保存前还会再次
   确认，避免因为数据重复导致选型结果不稳定。

## 二、怎么拿到 .exe 文件（通过 GitHub Actions 云端打包）

因为本程序最终要在 Windows 上以 .exe 形式运行，而 .exe 必须在 Windows
环境下打包才能生成，所以这里用 GitHub 提供的免费云端「Actions」功能来
自动打包——你只需要上传文件、点几次鼠标，不需要会写代码，也不需要自己
装 Python。整个过程大约 5 分钟。

### 第 1 步：注册/登录 GitHub
打开 https://github.com ，如果没有账号就免费注册一个（用邮箱即可）。

### 第 2 步：新建一个仓库（Repository）
1. 登录后点右上角 "+" → "New repository"。
2. 仓库名随便填，比如 `TE-AP-00`。
3. 选择 "Private"（私有，别人看不到）或 "Public" 都可以。
4. 不要勾选 "Add a README file"（我们已经准备好了完整文件）。
5. 点击绿色的 "Create repository"。

### 第 3 步：把本程序的所有文件上传上去
1. 新建好的仓库页面里会看到一个 "uploading an existing file" 的蓝色链接，点它。
   （或者在仓库页面点 "Add file" → "Upload files"）
2. 把你收到的 `TE-AP-00` 整个文件夹**里面的所有文件和文件夹**
   （包括隐藏的 `.github` 文件夹）一次性拖拽到网页上传区域。
   > 小技巧：如果浏览器不方便整体拖拽 `.github` 这种隐藏文件夹，
   > 可以把整个项目先打包成一个 zip，用 GitHub 网页的拖拽上传 zip 文件，
   > 上传后 GitHub 会提示是否要展开放置 —— 如果没有这个选项，
   > 最简单的办法是安装一次 [GitHub Desktop](https://desktop.github.com/)
   > 客户端，登录后选择 "Add local repository" 指向这个文件夹，
   > 点击 "Publish repository" 即可一次性把所有文件（含隐藏文件夹）传上去，
   > 全程鼠标点击，不需要敲命令行。
3. 在网页下方填写一句提交说明（比如"首次上传"），点击绿色的
   "Commit changes"。

### 第 4 步：查看自动打包结果
1. 上传完成后，点击仓库页面顶部的 "Actions" 标签。
2. 你会看到一个名为"打包 Windows exe"的流程正在运行（黄色圆点）
   或已完成（绿色勾）。第一次运行大约需要 2-4 分钟。
3. 等它变成绿色勾以后，点进这次运行记录，往下滚动到
   "Artifacts"（产出物）区域，会看到一个 `TE-AP-00-windows` 的下载链接，
   点击即可下载一个 zip 包。
4. 解压这个 zip，里面的 `TE-AP-00.exe` 双击即可运行
   （**注意要保留同一文件夹里的 `data` 子文件夹**，不要把 exe 单独拿出来，
   因为厂家数据、密码、设置都保存在 `data` 文件夹里）。

### 以后怎么更新
以后如果厂家数据或代码有修改，只需要把改动后的文件重新上传/提交到这个
GitHub 仓库（或用 GitHub Desktop 点一下 "Commit" + "Push"），
Actions 会自动重新打包，重复第 4 步下载最新的 zip 即可。

> 如果你更熟悉电脑操作，也可以跳过 GitHub，直接在一台 Windows 电脑上
> 双击运行项目里的 `build_exe.bat`，效果是一样的（见下方"本地打包"）。

## 三、日常使用说明

程序内置了完整的使用说明，可在菜单【帮助 - 程序说明】中查看，要点如下：

1. 表格左侧浅黄色【填写区】填写或从 Excel 粘贴阀门的"口径、压力等级、
   配置扭矩、上法兰尺寸、上连接轴尺寸、安全系数、气源压力"，点击
   【▶ 配置执行器】即可批量算出推荐型号，结果显示在右侧浅蓝色【输出区】。
   - 口径、压力等级只是用来标识这是哪条阀门（比如 DN50、PN16），不参与计算。
   - 法兰规范尺寸格式：`法兰/法兰`（小的写前面）或 `定制`。例：`F03/F05`
   - 上连接轴尺寸格式：`边长*边长*轴高` 或 `定制`。例：`17*17*25`
2. 若按扭矩选出的型号无法与阀门安装尺寸匹配，程序会自动就近匹配一款能
   安装的型号：被放大的型号会在结果文字后标注"（放大）"，被缩小的会标注
   "（缩小）"，对应行也会标黄提醒，请留意核实。
3. 双击表格左侧填写区的"阀门名称/口径/压力等级/配置扭矩/上法兰尺寸/
   上连接轴尺寸/安全系数/气源压力"这几列可以直接编辑；右侧输出区为只读。
4. 算完之后点【复制全部结果到剪贴板】，到 Excel 里 Ctrl+V 即可粘贴整张表
   （含表头；只会复制当前【显示列设置】里勾选显示的执行器类型）。
5. 【工具 - 数据来源设置】可以为 DA/SR/精小型电动/Q型电动/蜗轮 这 5 种
   执行器类型分别选择当前使用哪个厂家的数据——改完之后需要重新点一次
   【▶ 配置执行器】才会用新的数据重新计算。
6. 【工具 - 显示列设置】可以勾选只看哪几种执行器类型的结果，不需要的
   类型可以先隐藏，界面会更清爽。

### 密码说明
- 出厂默认管理员密码：`vatten2026`
- 首次使用建议立即在菜单【工具 - 修改管理员密码】中修改。
- 密码只在保存到本机的 `data/config.json` 中以加密哈希形式保存，
  不会以明文形式出现在任何文件里。

## 四、与原 Excel 程序的对比（完整透明说明）

为了让接手维护的人完全了解改动范围，这里如实列出所有"逻辑层面"的变化：

### 1）核心算法 —— 保持一致
- 扭矩匹配、安装尺寸匹配（含按型号顺序就近放大/缩小）、
  电动执行器多档连接轴尺寸变体匹配 等核心算法，已和原 VBA 代码
  （模块2：主程序/配置函数/安装匹配/对比法兰/对比连接轴/向上查找/
  向下查找）逐一对应移植，并用原 Excel 文件里已经算好的 21 行真实数据
  （DN15~DN200 等）做了自动化回归测试，全部一致（见 `tests/test_matching.py`）。

### 2）发现的两处原逻辑"细节问题"，本版选择按原样保留
- **离合器选型中的"按法兰尺寸数值判断大小离合器"分支**：原 VBA 用
  `Val(Split(VF,"/")(0))` 取阀门法兰字符串（如 "F07"）的"数值部分"，
  但 VBA 的 `Val()` 函数遇到非数字开头的字符串会直接返回 0，所以这个值
  在所有合法输入下恒为 0——这导致 DA100/DA140/DA190/DA240/SR100 这几档
  里"如果数值较大就选大离合器"的分支在当前输入规则下永远不会被触发。
  本版按"现有逻辑"原样移植（即同样恒选较小离合器），只在代码注释和本
  说明里如实记录，**没有自行更改判定结果**。如果确认这是疏漏并需要修正，
  可以联系我说明后另行调整。
- **"没有能安装的执行器"在真实数据下几乎不会出现**：因为各厂家数据表里
  最大几档型号几乎都用"定制"作为兜底（表示万能匹配），所以"就近放大"
  几乎总能在表的高端找到一个"定制"档位兜底。这不是bug，是原表数据本身
  的设计（最大型号留了"定制"出口），这里说明一下以免使用时有疑惑。

### 3）功能层面的新增（原 Excel 没有）
- 密码保护的执行器参数编辑器（需求3）
- 每种执行器类型可分别录入多个厂家数据，并独立选择数据来源（需求5）
- 批量从 Excel 粘贴导入 / 批量复制结果回 Excel（需求4，原来只能在
  Excel 表格内手动操作，现在改成跨程序的剪贴板交互）
- 自动备份：每次在编辑器里保存厂家数据前，会自动把修改前的版本备份到
  `data/actuator_types/_backup/`，避免误操作丢失数据。
- 型号名称、连接尺寸变体的自动查重提示。
- 口径、压力等级两个阀门识别列；输出区执行器类型可自由增减显示。

### 4）一次结构调整：厂家数据从"厂家优先"改为"执行器类型优先"
最初的版本是"先选厂家，厂家下面包含全部5种执行器类型数据"；根据实际
使用反馈（同一种执行器类型常常需要对比多个厂家的数据，而不是整个厂家
的全部产品线绑在一起切换），调整为"先选执行器类型，类型下面是各个
厂家的数据"，并支持 5 种类型分别独立选择当前用哪个厂家——这样更贴近
实际业务场景。原来按厂家存储的数据已经自动迁移到新结构，没有丢失任何
已录入的型号参数。法兰规范参照表（ISO5211最大扭矩）也因此从"每个厂家
各自一份"改为"全局共用一份"，因为它本质上是行业标准参照数据，
与具体哪个厂家无关。

### 5）界面/交互层面的调整
- 原 Excel 用单元格背景色（绿色=放大，红色=缩小）标记型号是否被
  就近调整；本程序改用结果文字后缀"（放大）"/"（缩小）"+整行浅色高亮
  来表达同样的信息。这是因为颜色信息无法通过纯文本的 Excel 粘贴（TSV）
  方式保留下来，改成文字标注后，即使把结果复制粘贴到一张新的 Excel
  表格里，提醒信息依然不会丢失，这是比原方式更稳妥的设计。
- 原 Excel 中"显示尺寸""偏差值"两项设置（在 UserForm1 设置面板里可见），
  经逐行核对 VBA 代码确认**并未接入任何实际计算逻辑**（设置了但没有任何
  地方读取使用），属于历史遗留的未生效选项。为避免误导用户以为它们会
  影响配置结果，本版的设置面板中默认不再展示这两项。
- 原 VBA 遇到某一行数据校验不通过（GSJC 报错）时，会用 `MsgBox` + `End`
  直接中断整张表的处理，导致这一行之后的所有行都不会被计算。本程序改为：
  记录该行的错误信息（显示在"备注"列），继续处理其余行，最后统一弹窗
  汇总告知哪些行有问题——这样不会因为一行填错就丢失其它行已经算好的结果。
- 原 Excel 中"格式化扭矩"这一列表头存在，但通查 VBA 代码后确认从未被
  写入或读取（纯粹的历史遗留空列），本版界面中已不再保留这一列。

以上调整均不影响任何一行有效输入数据的"计算结果"本身，只影响错误处理
方式、设置面板的展示内容、以及结果的呈现形式。

## 五、项目文件结构

```
TE-AP-00/
├── app.py                     # 程序入口
├── actuator_core/              # 核心逻辑（与界面无关，可独立测试）
│   ├── models.py               # 数据结构定义
│   ├── matching.py             # 选型算法（移植自原VBA模块2）
│   ├── actuator_type_store.py  # 执行器类型→厂家 数据的读写管理
│   ├── flange_spec_store.py    # 全局法兰规范参照表的读写管理
│   ├── auth.py                 # 密码保护
│   ├── settings_store.py       # 设置项的持久化（含每类型的厂家选择、显示列）
│   ├── version_history.py      # 版本记录
│   ├── clipboard_io.py         # Excel复制/粘贴格式转换
│   └── paths.py                # 路径定位（兼容直接运行/打包后运行）
├── ui/                          # 界面层（Tkinter）
│   ├── main_window.py
│   ├── grid.py                  # 主表格控件（左右两个色块的Treeview）
│   ├── editable_table.py        # 通用可编辑小表格控件（带自动查重）
│   ├── vendor_editor.py         # 厂家数据管理对话框（执行器类型优先）
│   └── dialogs.py               # 设置/数据来源/显示列/法兰规范/版本记录/
│                                 #   说明/密码 等对话框
├── data/                         # 数据文件（程序运行时会读写这里）
│   ├── actuator_types/           # 每种执行器类型一个文件，内含多个厂家数据
│   │   ├── DA.json
│   │   ├── SR.json
│   │   ├── J.json
│   │   ├── Q.json
│   │   └── WORM.json
│   ├── flange_spec.json          # 全局法兰规范参照表
│   ├── config.json               # 密码哈希 + 各类设置项
│   └── version_history.json      # 版本更新记录
├── tests/
│   └── test_matching.py          # 自动化回归测试
├── build_exe.bat                  # 本地一键打包脚本（Windows）
├── .github/workflows/
│   └── build-windows-exe.yml      # GitHub Actions 云端打包配置
└── requirements.txt
```
